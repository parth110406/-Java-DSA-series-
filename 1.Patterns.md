Approach
----
### STEP 1 — Understand the Problem


Before coding:
Ask yourself:

• What is given?
• What is asked?
• Array? String? Tree?
• Continuous subarray?
• Sorted array?

### STEP 2 — Identify the Pattern

### STEP 3 — Think Brute Force First

Never directly think optimal.

First think:
“What is the simplest possible solution?”
Even if complexity is bad.

### STEP 4 — Optimize Gradually

Ask:
• Can I reduce nested loops?
• Can HashMap help?
• Can pointers help?
• Can sliding window help?
• Can sorting help?

### STEP 5 — Dry Run on Paper

Very important.

Take:
• small input,
• trace variables,
• track pointers/windows/maps.
Most bugs disappear here.

### STEP 6 — Code Slowly & Cleanly
Do NOT speed code.

Write:
• meaningful variable names,
• small conditions,
• proper loops.

Bad coding causes more confusion than DSA itself.
